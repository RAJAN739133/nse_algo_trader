# ML Model Storage

## Where the model lives
- `stock_predictor.pkl` — Trained GradientBoosting model (~350KB)
- Generated by: `python -m data.train_pipeline demo` (or `train`)
- Used by: `live_paper.py`, `run_backtest.py`, `main.py`

## Model lifecycle
- **Train:** Weekly (Saturday 6 AM via scheduler, or manually)
- **Use:** Every trading day at 9:10 AM for stock scoring
- **Size:** ~350KB (small enough to commit to git)

## For deployment (Render/DigitalOcean)
The model is NOT in git (it's gitignored — generated locally).
On deploy, the system auto-trains on first startup:
1. Checks if `models/stock_predictor.pkl` exists
2. If not → auto-trains on synthetic data (takes 30 seconds)
3. Saturday cron job retrains weekly with real data via yfinance
4. Scoring works without model too (uses technical scores only, 70/100 max)
